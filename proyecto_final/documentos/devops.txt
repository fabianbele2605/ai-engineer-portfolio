Docker is a platform for building, shipping and running applications in containers.
Docker containers package code and dependencies together for consistent execution anywhere.
Dockerfile is a text file with instructions for building a Docker image.
Docker images are read-only templates used to create containers.
Docker Hub is a public registry for sharing and downloading Docker images.
Docker Compose defines and runs multi-container applications using a YAML file.
Docker volumes persist data generated by containers beyond the container lifecycle.
Docker networks allow containers to communicate with each other securely.
Kubernetes is an open-source system for automating deployment and scaling of containers.
Kubernetes pods are the smallest deployable units containing one or more containers.
Kubernetes deployments manage the desired state of pod replicas in a cluster.
Kubernetes services expose pods to network traffic inside or outside the cluster.
Kubernetes ingress manages external access to services in a cluster via HTTP/HTTPS.
CI/CD stands for Continuous Integration and Continuous Deployment.
GitHub Actions automates workflows for building, testing and deploying code.
Infrastructure as Code manages infrastructure using configuration files like Terraform.
Git is a distributed version control system for tracking changes in source code.
Git branches allow working on features independently without affecting the main codebase.
Git pull requests are proposals to merge changes from one branch into another.
DevOps combines development and operations practices to shorten the development lifecycle.
